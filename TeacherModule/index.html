Module #2: Loops

Prompt: Create a multi-color pattern of one shape that fills up the canvas.

A block of code that can be executed more than once in a row. When we see that a process has to repeat multiple times in a row, we write a loop. Loops allow us to create efficient code that automates processes to make scalable, manageable programs. They perform repetitive tasks so that we as programmers don’t need to code those tasks manually each time.

Every loop has three parts:
  1. Initialization: we set a starting condition for our loop here. This part is only executed once at the beginning of the loop.
  2. Condition: in each iteration of the loop, we check the truth value of the condition. If the condition is true, the loop continues to run; otherwise, we exit the loop.
  3. Update: each time the loop runs, we update our state (otherwise, nothing changes and we’ll be stuck in our loop forever!). Often, loops will increment or decrement a loop counter for the update step.

The for loop is a loop that does something a specified number of times. Here is what a for loop looks like: 

                        for (initialize; condition; update) { 
                            statements 
                        }
More concretely, we might have a for loop that looks like this: 

                        for (var i = 0; i < 3; i += 1) { 
                            print “hello world”
                        }
Here, we initialize a counter i, which increments by 1 every time the loop runs. Our loops runs as i is less than 3, and each time the loop repeats, the program prints out “hello world.” At then end, “hello world” will be printed three times in the output console. Watch out for infinite loops! If the condition always evaluates for true, then the loop never terminates. For example, if we don’t have an update, then the condition is never met: x = 1; while x < 5 {print x}
